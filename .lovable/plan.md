# /project route — real ODP data in Workspace UI

Add a new route `/project?app=<appNumber>&template=<templateCode>` that renders the same dark-themed Workspace shown in the screenshot, but populated from live USPTO ODP data instead of the canned samples in `src/components/demo/data.ts`. The existing `/demo` and its mock data stay untouched.

## Behavior

- Read `app` and `template` from URL search params (both optional).
- Fetch real data from USPTO ODP via existing server functions:
  - `fetchUsptoApplication({ data: { applicationNumber } })` → metadata (title, assignee, inventors, art unit, examiner, filing date).
  - `fetchUsptoTransactions({ data: { applicationNumber } })` → transactions list.
- Map ODP responses into the existing `AppData` shape consumed by `Workspace`.
- Run `detectEvent(transactions)` on the real transactions.
  - If detection returns an event → use it.
  - If detection returns nothing AND `template` param is a known code → use `template` as fallback event code.
  - Otherwise render the empty/"no docketable event" state.
- Missing/invalid `app` param → show a small form to enter one and update the URL.
- Loading state → skeleton of the sidebar + main area. Errors → inline card with retry.
- Citations default to `[]` (ODP doesn't provide them here); Citation tool tab still renders and lets you add manually.

## Files

**New**
- `src/routes/project.tsx` — TanStack route with `validateSearch` (zod: `app?`, `template?`), `head()` metadata, component that runs both server fns via `useQuery`, maps into `AppData`, and renders `<Workspace app={...} onChangeApp={...} />`. `onChangeApp` opens an inline input that navigates with new `app` search param.
- `src/lib/uspto-mapping.ts` — pure helper `toAppData(apiData, txData, appNumber): AppData` that pulls `patentFileWrapperDataBag[0].applicationMetaData` fields (inventionTitle, applicantBag/firstApplicantName, inventorBag, groupArtUnitNumber, examinerNameText/firstInventorToFileIndicator, filingDate) and maps `eventDataBag` (or equivalent transactions array) to `{ code, description, date }[]` sorted desc. Uses `TRANSACTION_DESCRIPTIONS` from `demo/data.ts` to fill descriptions where ODP gives only a code.

**Edited**
- `src/routeTree.gen.ts` — auto-regenerated by the router plugin; no manual edit.

## Technical notes

- Route path: `createFileRoute("/project")` with `validateSearch: z.object({ app: z.string().optional(), template: z.string().optional() })`.
- Data fetching from the component (not the loader) via `useQuery` keyed on `["uspto-app", app]` and `["uspto-tx", app]`, enabled only when `app` is set. This avoids running protected/expensive server calls during SSR prerender.
- Reuse the existing `Workspace` component unchanged. It already accepts an `AppData` prop and internally calls `detectEvent`. To honor the `template` fallback without forking `Workspace`, `toAppData` will, when `template` is provided and no detectable event exists in real transactions, append a synthetic transaction with that code dated today so `detectEvent` picks it up — kept behind a comment noting it's a demo hint.
- SEO `head()`: title "Project — FenixAI", description referencing live USPTO workflow view; no og:image.
- No auth, no DB writes — this is a public shareable view.

## Out of scope

- Modifying `/demo` or `src/components/demo/data.ts`' sample data.
- Persisting projects to the database.
- Editing the auto-generated route tree by hand.
